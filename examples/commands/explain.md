---
name: explain
description: Explain a file or function in plain English
---

The user wants to understand code. Use $ARGUMENTS to identify what they want explained.

1. Read the file or function
2. Explain what it does in plain English, no jargon
3. Walk through the key logic step by step
4. Note any non-obvious patterns or clever tricks
5. If there are dependencies, briefly explain what they provide

Keep it conversational. Assume the reader is technical but unfamiliar with this specific codebase.
